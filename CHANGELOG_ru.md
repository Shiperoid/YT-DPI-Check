# YT-DPI v2.2.3

По сравнению с **v2.2.2** в этой версии в первую очередь обновлена bash-версия и сопутствующие материалы проекта.

### Bash (`YT-DPI.sh` — Git Bash, Linux, Entware)

* **Вердикт THROTTLED:** отдельный жёлтый статус, когда по TLS видно частичное ограничение (типичный сценарий «троттлинга» вместо полного блока).
* **Паритет с Windows-версией:** те же цели проверки, логика настроек и поведение интерфейса, что у актуального `YT-DPI.bat`.
* **IPv6:** более надёжная детекция и расширенная колонка таблицы под длинные адреса.
* **Latency и анимация:** исправлен расчёт задержки, снижены лаги анимации строки LAT в Git Bash.
* **Entware / OpenWrt:** совместимый способ запуска и приведение комментариев в скрипте к нормальному виду.
* **Режимы TLS:** при успешном HTTP учитывается выбранный режим TLS 1.2 / 1.3 / авто при выводе итогового вердикта.

### Windows (`YT-DPI.bat`)

* Номер версии **2.2.3** и сопутствующие правки стабильности и отображения (в том числе корректная работа с кириллицей и символами Unicode в консоли Windows PowerShell 5.1 при типовых настройках кодировки).

### Проект

* Обновления **README**, ссылка на поддержку автора, мелкие исправления геоданных провайдера (ISP).

# YT-DPI v2.0

В версии 2.0 утилита полностью переписана. Инструмент переведен с последовательного сканирования на многопоточную архитектуру, а также получил полностью новый интерактивный консольный интерфейс. Основной упор в обновлении сделан на скорость работы, детализацию тестируемых протоколов и анализ сети в реальном времени.

### Основные нововведения

* **Многопоточный движок (RunspacePools):** Сканирование больше не выполняется последовательно. Утилита одновременно запускает до 20 независимых фоновых потоков, что сократило общее время проверки всех доменов с ~30 до 2-3 секунд.
* **Детальное разделение протоколов:** Вместо объединения всего TLS-трафика в один результат, теперь отдельно тестируются **открытый HTTP (порт 80)**, **TLS 1.2 (порт 443)** и **TLS 1.3 (порт 443)**. Это позволяет точно определить, на каком уровне шифрования ТСПУ обрывает соединение.
* **Детекция тихих дропов (DRP):** Добавлено точное распознавание ситуаций, когда DPI или файрвол отбрасывает пакеты (blackhole) без отправки ответа. Теперь утилита отличает зависшее по таймауту TLS-рукопожатие (`DRP`) от активного сброса соединения (`RST`).
* **Новый интерактивный интерфейс:** Статичный дашборд с абсолютным позиционированием курсора (без прокрутки консоли). Во время работы отображаются динамические эффекты сканирования: поочередный подбор октетов IP-адреса, перебор Hex-байтов на тестируемых портах и изменяющийся пинг.
* **Живая панель телеметрии:** В правом верхнем углу добавлена панель, которая в реальном времени (в процессе сканирования) обновляет количество активных потоков (JOBS), потребление ОЗУ (RAM) и счетчики: заблокированные домены (BLOCKS), сброшенные пакеты (RST) и успешные подключения (CLEAN).
* **Расширенная сетевая аналитика:** Перед каждым сканированием утилита автоматически определяет и выводит на экран текущий активный **DNS**, **имя провайдера (ISP)**, **страну/город** и ближайший **узел Google CDN**.

### Улучшения и исправления

* **Мгновенная отмена сканирования:** Обработка нажатий `Q` или `ESC` теперь использует асинхронную остановку потоков (`BeginStop`). Интерфейс разблокируется моментально, не дожидаясь сетевых таймаутов зависших соединений.
* **Защита от зависаний консоли:** При запуске утилита через вызов C# API автоматически отключает «Режим быстрого редактирования» (QuickEdit Mode) в Windows. Теперь случайное выделение текста мышью в консоли не будет ставить выполнение скрипта на паузу.
* **Умное определение ОС для TLS 1.3:** Windows 7 и старые сборки Windows 10 не поддерживают TLS 1.3 на уровне системы. Утилита теперь распознает это ограничение ОС и выводит статус `N/A` (Недоступно) вместо ложного срабатывания `RST` (Блокировка).
* **Встроенная справка:** Добавлена кнопка `[H]`, открывающая мини-гайд с расшифровкой статус-кодов (OK, RST, DRP, N/A, FAIL) и вердиктов прямо внутри приложения, без необходимости переходить в браузер.
* **Устранено моргание экрана:** Механизм рендеринга интерфейса оптимизирован. Данные обновляются поверх старых значений без полной очистки консоли (`Clear-Host`), что убрало неприятное мерцание при повторных запусках сканирования.

### Расшифровка статус-кодов
* **OK:** Соединение установлено, TLS-рукопожатие завершено успешно.
* **RST:** Соединение принудительно разорвано (TCP Reset) оборудованием DPI/ТСПУ.
* **DRP:** Соединение "тихо" отброшено (Blackhole), TLS-рукопожатие прервано по таймауту.
* **N/A:** Протокол не поддерживается вашей ОС (например, TLS 1.3 на Windows 7).
* **--** / **FAIL:** Ошибка сокета или полная недоступность IP-адреса.

# YT-DPI v2.1.3

Версия 2.1.3 превращает утилиту из простого многопоточного сканера в профессиональный инструмент сетевой аналитики. Основной упор сделан на сохранение состояния, продвинутую работу с прокси и мгновенный запуск.

### 🚀 Производительность и Движок
*   **Система постоянного кэширования:** Все данные сети (Провайдер, CDN, DNS-записи) теперь сохраняются в `%LOCALAPPDATA%`. Повторные запуски происходят **мгновенно** (< 1 сек), так как интерфейс отрисовывается из кэша.
*   **Тайминги на основе Ticks:** Переход на хранение времени в системных тиках (целые числа). Это полностью решило проблему ошибок парсинга даты на разных версиях Windows и локалях.
*   **Отрисовка без мерцания:** Логика UI переписана на "один проход". Раздражающее моргание экрана при старте сканирования полностью устранено.
*   **Умный Dual-Stack (IPv6):** Добавлена полноценная поддержка IPv6. Скрипт определяет наличие IPv6-интернета и приоритезирует его, позволяя найти провайдеров, которые блокируют YouTube только на IPv4.

### 🛡️ Работа с Прокси и Аналитика
*   **Универсальный прокси-движок:** Полная поддержка **SOCKS5** и **HTTP** прокси с поддержкой авторизации.
*   **Автоопределение протокола:** Вам больше не нужно указывать тип прокси. Просто введите `IP:PORT` (например, `127.0.0.1:1080`), и скрипт сам подберет нужный метод рукопожатия.
*   **Честная диагностика через прокси:** Исправлен баг "ложноположительных" зеленых результатов при работе через прокси. Теперь TLS-проверка проводится максимально строго — вы увидите реальную картину работы вашего средства обхода.
*   **Тестер прокси:** По нажатию `[T]` запускается пошаговая диагностика: от доступности хоста до замера задержки (Latency).

### 🛠️ Стабильность и Инструменты
*   **Переработанный Апдейтер:** Новый механизм обновления через изолированные временные процессы. Это исключает ошибки "отказано в доступе" и гарантирует корректную замену файла.
*   **Deep Trace (L4 Traceroute):** Продвинутая трассировка на уровне TCP. Позволяет найти "точку цензуры" (ТСПУ/DPI) на маршруте через инкремент TTL.
*   **Потокобезопасный DNS-кэш:** Найденные IP-адреса синхронизируются между всеми 20+ потоками. Это сводит нагрузку на системный DNS к нулю при повторных проверках.
*   **Техническая документация:** Встроенная справка `[H]` превращена в полноценное руководство, объясняющее работу QUIC, SNI, шифрования Kyber и GGC.
