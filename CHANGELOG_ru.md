# YT-DPI v2.3.1 (hotfix)

**Сравнение в Git:** [2.3.0…2.3.1](https://github.com/Shiperoid/YT-DPI/compare/2.3.0...2.3.1) (после публикации тега `2.3.1`).

### Исправлено

* **Windows PowerShell 5.1 (`powershell.exe`):** встроенный C# для TLS‑сканера и трассировки снова **компилируется через `Add-Type`**. В 2.3.0 использовались конструкции и API, которые старый компилятор/рантайм 5.1 не принимает (`Random.Shared`, интерполяция `$"..."`, `?.` и др.) — из‑за этого у части пользователей в статусе было **«TLS scanner failed to load»**, а скан по **Enter** фактически не запускался. В 2.3.1 код приведён к совместимости с **Desktop 5.1** и проверен под **pwsh**.
* **Процесс релиза:** добавлены **`tools/release-gate.ps1`** (извлечение here-string из `YT-DPI.ps1` + компиляция в чистых процессах **5.1** и **Core**) и GitHub Actions workflow **`release-gate`**, чтобы такая регрессия не проходила незамеченной.

### Как обновиться с 2.3.0

* Запускайте **`YT-DPI.bat`** (он задаёт **`-ExecutionPolicy Bypass`**). Если открываете только **`YT-DPI.ps1`**, при строгой политике подписи Windows может отказать в запуске — тогда снова **`.bat`** или вручную: `powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\YT-DPI.ps1`.
* В программе нажмите **`[U]` UPDATE**, если GitHub уже показывает релиз **2.3.1** новее вашей версии (проверка идёт через API **последнего релиза**; файлы подтягиваются с ветки **`master`** — важно, чтобы туда уже попал хотфикс).
* Либо скачайте **`YT-DPI.ps1`** и при необходимости **`YT-DPI.bat`** со [страницы Releases](https://github.com/Shiperoid/YT-DPI/releases) и положите в одну папку.

### Текст для вставки в описание GitHub Release v2.3.1

Извиняемся за неудобства: в **2.3.0** на части компьютеров с **Windows PowerShell 5.1** не загружался C#‑движок TLS/трассировки, из‑за чего появлялась ошибка в статусной строке и скан по **Enter** не доходил до проверок.

**2.3.1** — исправление совместимости встроенного C# с **PowerShell 5.1** (и проверка на **PowerShell 7**). Рекомендуем обновиться: **`YT-DPI.bat`** или пункт **`[U]`** после появления релиза на GitHub, либо ручная замена **`YT-DPI.ps1`** / пары **`.bat` + `.ps1`** из Releases.

# YT-DPI v2.3.0

**Сравнение в Git:** [2.2.3…2.3.0](https://github.com/Shiperoid/YT-DPI/compare/2.2.3...2.3.0) (после публикации тега `2.3.0`; до тега см. [2.2.3…master](https://github.com/Shiperoid/YT-DPI/compare/2.2.3...master)).

Дальше по разделу **v2.3.0** изменения сгруппированы по смыслу: **добавлено**, **обновлено**, **исправлено**, **быстрее и меньше нагрузки на консоль**, затем отладка, проект и релиз.

### Упаковка и запуск (Windows)

* **Лаунчер + `YT-DPI.ps1`:** монолитный огромный `YT-DPI.bat` убран из основной поставки. **`YT-DPI.bat`** только вызывает `pwsh` / `powershell` и при отсутствии рядом **`YT-DPI.ps1`** **однократно** подтягивает его с GitHub (проверка размера). **Зачем:** меньше ложных срабатываний антивирусов и проще выкладывать обновления логики без пересборки «батника на тысячи строк».
* **Обновление [U]:** обновляется файл, с которого запустились (`.bat` или `.ps1`), с проверкой целостности; после успеха докачивается **вторая** часть комплекта рядом, чтобы не остаться с половиной набора.

### Добавлено

* **Ленивая загрузка C#:** TLS‑сканер и код трассировки подгружаются **перед сканом** или **перед Deep Trace**, а не всегда при старте скрипта.
* **Проверка интернета и подготовка скана:** перед запуском скана — быстрый тест доступности сети и подготовка целей; в статус-баре отображаются этапы («проверка сети», загрузка движка и т.п.).
* **Режим TLS в скане:** в настройках **[S]** пункт **3** — цикл **Auto → только TLS 1.2 → только TLS 1.3**; значение передаётся в воркеры (**`TlsMode`**).
* **Отладочный лог из UI:** пункты **[S]** **4–5** — запись в **`YT-DPI_Debug.log`** и опционально полные идентификаторы в заголовке; переменные **`YT_DPI_DEBUG`** и **`YT_DPI_DEBUG_IDENTIFIERS`**; **глобальный mutex** на файл и безопасная запись из **runspace’ов** через общее имя mutex.
* **Меню прокси [P]:** навигация **цифрами**, выход **0** / **Esc**, ввод адреса **строкой** с отменой **Esc**, подтверждение очистки истории **Y/N**, перерисовка при **ресайзе** без залипания ввода.
* **Справка [H]:** **пять** страниц, временное расширение буфера консоли и восстановление высоты после выхода; листание **N** / **P** и стрелками; выход по **Enter** / **Esc** или по **любой другой клавише** (удобно при неверной раскладке).
* **Сводка в статус-баре:** при отсутствии отдельного сообщения показывается **итог последнего скана** по числу вердиктов (AVAILABLE, THROTTLED, DPI, IP BLOCK и т.д.).
* **Прогресс скана в статус-баре:** полоска процента во время сбора результатов и фазы раскрытия (совместно с троттлингом — см. раздел про производительность).
* **Частичный «водопад»:** при повторном скане обновление в основном изменившихся строк и колонки LAT; выборочная перерисовка через **`Write-ResultLine`** / **`Write-ResultLatency`**.
* **`SynchronousProgress` в Deep Trace:** синхронная отдача текста прогресса вместо **`System.Progress`** в условиях pw7 без подходящего **`SynchronizationContext`**.
* **Эвристика «ложного IPv6»:** при **HasIPv6** и полном наборе строк **IP BLOCK** — сброс флага IPv6 и удаление IPv6 из DNS‑кэша.
* **Меню [S]:** пункт **1–2** (как ранее: приоритет IP, сброс кэша) дополнены описанными выше пунктами **3–5**; ввод с **`Read-MenuKeyOrResize`** при ресайзе.
* **Bash (`YT-DPI.sh`):** те же **физические** клавиши при **русской** раскладке, что и латинские пункты меню (в т.ч. при прерывании скана).

### Обновлено

* **Главный экран:** там, где нужно, **`Draw-UI`** с **`ClearScreen=$false`** — перерисовка **поверх** ячеек без полного **`[Console]::Clear`**, меньше мерцания.
* **Ресайз:** раздельная логика для простоя и **фазы скана** (порог > 1 колонка/строка), меньше ложных полных перерисовок из‑за дрожания **`WindowWidth`** / **`Height`**.
* **Старт консоли:** UTF-8, скрытый курсор, уменьшенный **`CursorSize`**.
* **Главный цикл:** раз в секунду подтягивается **NetInfo** после фонового job **без блокировки** ожидания клавиши.
* **Фаза сбора скана:** вместо анимации **каждой** строки каждый кадр (v2.2.3) — прогресс в статус-баре и обновление строк по завершении воркеров.
* **Таблица:** динамические позиции колонок и пересчёт ширины IP после скана для длинных **IPv6**.
* **Пул Runspace:** без изменения модели — **8…24** воркеров (`min(CPU×3, 24)`), с прокси потолок **12**, не больше числа целей.

### Исправлено

* **`Draw-UI` и тип результата:** если в третий параметр передали **boolean** вместо массива результатов, выполняется сброс в **`$null`** — раньше ломалась **только первая** строка таблицы.
* **Deep Trace в PowerShell 7:** снижен риск падений при выводе прогресса из фоновых потоков (переход на **`SynchronousProgress`**).
* **Прокси:** после **неудачного** полного теста конфигурация **восстанавливается** из снимка — не остаётесь с нерабочим прокси.
* **Диагностика прокси:** поэтапные таймауты и сообщения (**`Wait-TcpBeginConnectWithStatus`** и связанная логика).

### Быстрее и меньше нагрузки на консоль

* **Холодный старт:** отложенный **`Add-Type`** до первого использования TLS / трассировки.
* **Фаза сбора скана:** нет постоянной перерисовки LAT и индикатора «SCANNING» по всем строкам на каждом кадре.
* **Троттлинг статус-бара:** ограничение частоты обновления строки прогресса по времени и «ведрам» процента.
* **Кеш `Draw-StatusBar`:** нижние строки перерисовываются только при смене содержимого (**`StatusFeedbackCacheKey`** / **`StatusControlsCacheKey`**).
* **Частичное обновление строк результатов:** **`Write-ResultLatency`** и выборочный **`Write-ResultLine`** уменьшают число операций записи при повторных сканах.

### Отладка и приватность

* По умолчанию в заголовке лога **обезличены** пути и часть метаданных; развернутый заголовок — пункт **[S]** **5** или **`YT_DPI_DEBUG_IDENTIFIERS`**.
* **Ротация** лога при превышении размера.

### Проект

* Вспомогательные скрипты в **`tools/`**; **`img/`**, **`tools/YT-DPI-algorithm`**, **`.editorconfig`** / **`.gitattributes`** по мере развития репозитория.

### Релиз на GitHub (v2.3.0)

Для Windows скачайте **`YT-DPI.bat`** и **`YT-DPI.ps1`** в **одну папку**, запускайте **`YT-DPI.bat`**. Для Linux / macOS / Git Bash / Entware — **`YT-DPI.sh`**.

# YT-DPI v2.0

В версии 2.0 утилита полностью переписана. Инструмент переведен с последовательного сканирования на многопоточную архитектуру, а также получил полностью новый интерактивный консольный интерфейс. Основной упор в обновлении сделан на скорость работы, детализацию тестируемых протоколов и анализ сети в реальном времени.

### Основные нововведения

* **Многопоточный движок (RunspacePools):** Сканирование больше не выполняется последовательно. Утилита одновременно запускает до 20 независимых фоновых потоков, что сократило общее время проверки всех доменов с ~30 до 2-3 секунд.
* **Детальное разделение протоколов:** Вместо объединения всего TLS-трафика в один результат, теперь отдельно тестируются **открытый HTTP (порт 80)**, **TLS 1.2 (порт 443)** и **TLS 1.3 (порт 443)**. Это позволяет точно определить, на каком уровне шифрования ТСПУ обрывает соединение.
* **Детекция тихих дропов (DRP):** Добавлено точное распознавание ситуаций, когда DPI или файрвол отбрасывает пакеты (blackhole) без отправки ответа. Теперь утилита отличает зависшее по таймауту TLS-рукопожатие (`DRP`) от активного сброса соединения (`RST`).
* **Новый интерактивный интерфейс:** Статичный дашборд с абсолютным позиционированием курсора (без прокрутки консоли). Во время работы отображаются динамические эффекты сканирования: поочередный подбор октетов IP-адреса, перебор Hex-байтов на тестируемых портах и изменяющийся пинг.
* **Живая панель телеметрии:** В правом верхнем углу добавлена панель, которая в реальном времени (в процессе сканирования) обновляет количество активных потоков (JOBS), потребление ОЗУ (RAM) и счетчики: заблокированные домены (BLOCKS), сброшенные пакеты (RST) и успешные подключения (CLEAN).
* **Расширенная сетевая аналитика:** Перед каждым сканированием утилита автоматически определяет и выводит на экран текущий активный **DNS**, **имя провайдера (ISP)**, **страну/город** и ближайший **узел Google CDN**.

### Улучшения и исправления

* **Мгновенная отмена сканирования:** Обработка нажатий `Q` или `ESC` теперь использует асинхронную остановку потоков (`BeginStop`). Интерфейс разблокируется моментально, не дожидаясь сетевых таймаутов зависших соединений.
* **Защита от зависаний консоли:** При запуске утилита через вызов C# API автоматически отключает «Режим быстрого редактирования» (QuickEdit Mode) в Windows. Теперь случайное выделение текста мышью в консоли не будет ставить выполнение скрипта на паузу.
* **Умное определение ОС для TLS 1.3:** Windows 7 и старые сборки Windows 10 не поддерживают TLS 1.3 на уровне системы. Утилита теперь распознает это ограничение ОС и выводит статус `N/A` (Недоступно) вместо ложного срабатывания `RST` (Блокировка).
* **Встроенная справка:** Добавлена кнопка `[H]`, открывающая мини-гайд с расшифровкой статус-кодов (OK, RST, DRP, N/A, FAIL) и вердиктов прямо внутри приложения, без необходимости переходить в браузер.
* **Устранено моргание экрана:** Механизм рендеринга интерфейса оптимизирован. Данные обновляются поверх старых значений без полной очистки консоли (`Clear-Host`), что убрало неприятное мерцание при повторных запусках сканирования.

### Расшифровка статус-кодов
* **OK:** Соединение установлено, TLS-рукопожатие завершено успешно.
* **RST:** Соединение принудительно разорвано (TCP Reset) оборудованием DPI/ТСПУ.
* **DRP:** Соединение "тихо" отброшено (Blackhole), TLS-рукопожатие прервано по таймауту.
* **N/A:** Протокол не поддерживается вашей ОС (например, TLS 1.3 на Windows 7).
* **--** / **FAIL:** Ошибка сокета или полная недоступность IP-адреса.

# YT-DPI v2.1.3

Версия 2.1.3 превращает утилиту из простого многопоточного сканера в профессиональный инструмент сетевой аналитики. Основной упор сделан на сохранение состояния, продвинутую работу с прокси и мгновенный запуск.

### 🚀 Производительность и Движок
*   **Система постоянного кэширования:** Все данные сети (Провайдер, CDN, DNS-записи) теперь сохраняются в `%LOCALAPPDATA%`. Повторные запуски происходят **мгновенно** (< 1 сек), так как интерфейс отрисовывается из кэша.
*   **Тайминги на основе Ticks:** Переход на хранение времени в системных тиках (целые числа). Это полностью решило проблему ошибок парсинга даты на разных версиях Windows и локалях.
*   **Отрисовка без мерцания:** Логика UI переписана на "один проход". Раздражающее моргание экрана при старте сканирования полностью устранено.
*   **Умный Dual-Stack (IPv6):** Добавлена полноценная поддержка IPv6. Скрипт определяет наличие IPv6-интернета и приоритезирует его, позволяя найти провайдеров, которые блокируют YouTube только на IPv4.

### 🛡️ Работа с Прокси и Аналитика
*   **Универсальный прокси-движок:** Полная поддержка **SOCKS5** и **HTTP** прокси с поддержкой авторизации.
*   **Автоопределение протокола:** Вам больше не нужно указывать тип прокси. Просто введите `IP:PORT` (например, `127.0.0.1:1080`), и скрипт сам подберет нужный метод рукопожатия.
*   **Честная диагностика через прокси:** Исправлен баг "ложноположительных" зеленых результатов при работе через прокси. Теперь TLS-проверка проводится максимально строго — вы увидите реальную картину работы вашего средства обхода.
*   **Тестер прокси:** По нажатию `[T]` запускается пошаговая диагностика: от доступности хоста до замера задержки (Latency).

### 🛠️ Стабильность и Инструменты
*   **Переработанный Апдейтер:** Новый механизм обновления через изолированные временные процессы. Это исключает ошибки "отказано в доступе" и гарантирует корректную замену файла.
*   **Deep Trace (L4 Traceroute):** Продвинутая трассировка на уровне TCP. Позволяет найти "точку цензуры" (ТСПУ/DPI) на маршруте через инкремент TTL.
*   **Потокобезопасный DNS-кэш:** Найденные IP-адреса синхронизируются между всеми 20+ потоками. Это сводит нагрузку на системный DNS к нулю при повторных проверках.
*   **Техническая документация:** Встроенная справка `[H]` превращена в полноценное руководство, объясняющее работу QUIC, SNI, шифрования Kyber и GGC.

1. Download **`YT-DPI.bat`** / Скачайте файл.
2. Run (No Admin required) / Запустите (права админа не требуются, но рекомендуются для Deep Trace).
3. Press `[ENTER]` to Scan.
