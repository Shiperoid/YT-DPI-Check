# Политика ветки `feature/terminal-gui` и слияния в `master`

## Содержимое ветки

- Корневой [`README.md`](../README.md) в этой ветке описывает **только** превью Terminal.Gui + `YT-DPI.Core` / `YT-DPI.App` (без инструкций по другим линиям поставки).
- Остальные файлы репозитория (скрипты, релизные workflow и т.д.) могут существовать параллельно; их судьба при merge определяется отдельно.

## При merge в `master`

1. **README:** либо восстановить полное описание продукта в корневом `README.md`, либо оставить краткий указатель и вынести описание превью в отдельный файл (например `docs/terminal-gui-readme.md`), либо объединить два раздела в одном README — **решение мейнтейнера**.
2. **Конфликты:** разрешать с приоритетом стабильности основной линии; изменения в `YT-DPI.Core` / App не должны ломать существующие проверки без явного обновления workflow.
3. **Релизы:** публикация артефакта превью из `terminal-gui-build` не обязана совпадать с тегами релиза основного продукта; при необходимости завести отдельный pre-release или имя артефакта с суффиксом `-gui-preview`. Отдельно: workflow **`core-ps-bundle`** на `master` — zip **`YT-DPI.Core.dll`** + **`YT-DPI.ps1`** (см. [`bundle-core-ps.md`](bundle-core-ps.md)).
