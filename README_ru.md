### Обзор
**YT-DPI Check** — это легковесная диагностическая утилита без зависимостей, предназначенная для определения причин проблем с доступом к YouTube и сервисам Google. Инструмент выполняет глубокий анализ сетевых протоколов (TLS SNI Handshake и QUIC), чтобы отличить обычные сбои соединения от активного вмешательства систем DPI (ТСПУ).

### Системные требования
*   **ОС:** Windows 10 или Windows 11 (рекомендуется).
*   **Устаревшие ОС:** Windows 7 / 8.1 поддерживаются, но требуют установленных **PowerShell 5.1** и **.NET Framework 4.5+**.
*   **Окружение:** PowerShell 5.1 или выше.

### Важные примечания
*   **Ограничение TLS 1.3:** Windows 7 не поддерживает TLS 1.3 на системном уровне. Так как современные системы ТСПУ/DPI часто нацелены именно на TLS 1.3, проверка на Windows 7 может показать статус **AVAILABLE** даже если сервис заблокирован для пользователей Windows 10/11. Это происходит из-за автоматического отката системы на старый протокол TLS 1.2.

### Основные возможности
*   **Анализ QUIC/UDP:** Проверка порта UDP 443 на предмет блокировки или замедления современного протокола QUIC.
*   **Тест SNI Handshake:** Проверка поля "Server Name Indication" для обнаружения фильтров, которые срабатывают только при попытке зайти на конкретный ресурс.
*   **Детекция активного сброса (Reset):** Определяет, когда ТСПУ принудительно разрывает соединение, отправляя пакеты `TCP RST`.
*   **Автопоиск CDN:** Автоматическое обнаружение ближайшего узла Google Global Cache (GGC) для тестирования прямого видеопотока.
*   **Полный охват:** Тестирование основного интерфейса, видеосерверов (googlevideo) и критически важных API.

### Как использовать
1.  Скачайте файл **`YT-DPI-Check.bat`** из раздела [последнего релиза](../../releases/latest).
2.  Запустите его двойным щелчком. **Права администратора не требуются.**
3.  Ознакомьтесь с таблицей результатов и итоговым разделом **DIAGNOSIS**.

### Подробное руководство по статусам

#### Колонки: TCP / TLS
*   **OK**: Успешно. Соединение установлено, TLS-рукопожатие завершено.
*   **FL (Failed)**: Порт закрыт или сервер полностью недоступен на сетевом уровне.
*   **DR (Dropped)**: **[Признак DPI]** Пакет отправлен, но сервер «исчез». DPI идентифицировал SNI и «тихо» отбросил ваши пакеты.
*   **RS (Reset)**: **[Признак DPI]** Соединение установлено, но как только было передано имя сайта, ТСПУ принудительно разорвал связь.
*   **--**: Пропущено из-за ошибки на предыдущем этапе (например, DNS).

#### Колонка: UDP
*   **OK**: Порт открыт, протокол QUIC потенциально доступен.
*   **BK (Blocked)**: Провайдер явно блокирует UDP-трафик на 443 порту (часто используется для замедления YouTube).
*   **??**: Состояние неизвестно или время ожидания истекло.

#### Колонка: RESULT (Вердикт)
| Вердикт | Значение |
| :--- | :--- |
| **AVAILABLE** | Сервис работает нормально. |
| **DPI BLOCK** | Обнаружена активная фильтрация SNI (блокировка на уровне DPI). |
| **IP BLOCK** | IP-адрес недоступен (блокировка по IP). |
| **DNS ERROR** | DNS-провайдер выдает неверные данные или не может разрешить имя домена. |

### Важные примечания
*   **Антивирусы:** Некоторые антивирусы могут помечать скрипт как «сетевой инструмент», так как он выполняет ручные проверки соединений. Скрипт имеет открытый исходный код, доступный для аудита.
*   **VPN/Прокси:** Если у вас включен VPN или средства обхода (например, GoodbyeDPI), скрипт покажет статус **AVAILABLE**, так как проверка пройдет через них. Для тестирования реального состояния сети провайдера их нужно отключить.
